# myRPC

这是一个面向 RPC 学习路线的练手项目。

第一阶段目标很简单：先把最基础的 TCP 网络通信跑起来。

## 当前能力

- 监听指定端口
- 接收客户端连接
- 读取客户端发送的数据
- 回写一段固定响应

## 构建

```bash
cmake -S . -B build
cmake --build build
```

## 运行

```bash
./build/simple_tcp_server
```

默认监听 `127.0.0.1:8080`。

你可以用 `nc` 测试：

```bash
printf 'hello\n' | nc 127.0.0.1 8080
```

## 下一步建议

1. 把当前阻塞式单连接模型改成“循环处理多个连接”。
2. 再往后升级到 `select` / `poll` / `epoll`。
3. 最后再逐步抽象出 RPC 所需的协议编解码、连接管理和线程模型。
