# myRPC

这是一个面向 RPC 学习路线的练手项目。

第一阶段目标很简单：先把最基础的 TCP 网络通信跑起来。

## 当前能力

- 监听指定端口
- 基于 `epoll` 的单线程 IO 多路复用
- 并发处理多个客户端连接
- 在同一个客户端连接内反复读取多条消息
- 按行回写响应，客户端断开前不会主动关闭连接

## 构建

```bash
cmake -S . -B build
cmake --build build
```

## 运行

```bash
./build/simple_tcp_server
```

默认监听 `127.0.0.1:8080`。

你可以用 `nc` 测试：

```bash
nc 127.0.0.1 8080
```

连上后连续输入多行内容，每行都会收到一条响应：

```text
hello
ok: hello
world
ok: world
```

## 下一步建议

1. 抽象请求/响应结构，替换掉当前“按行文本协议”。
2. 再逐步引入 RPC 所需的编解码、连接管理和线程模型。
3. 在事件循环上补充空闲连接清理、限流和更完整的错误处理。
